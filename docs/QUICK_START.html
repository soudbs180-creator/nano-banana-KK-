🚀 KK Studio Supabase 配置

3 分钟完成数据库设置

1 打开 SQL Editor

点击下方按钮打开 Supabase SQL Editor：


打开 SQL Editor

如果提示登录，请使用您的 Supabase 账号登录

2 创建新查询

在 SQL Editor 页面中：

  • 点击左侧 "+ New Query" 按钮
  • 或者点击右上角的 "New Query"

3 复制并执行 SQL

点击下方按钮复制完整 SQL：



然后粘贴到 SQL Editor 中，点击 "Run"

-- ============================================ -- KK Studio Complete Supabase Setup -- ============================================ -- 1. Profiles Table (用户资料 + 积分) CREATE TABLE IF NOT EXISTS public.profiles ( id UUID PRIMARY KEY REFERENCES auth.users(id) ON DELETE CASCADE, email TEXT, credits DECIMAL DEFAULT 0, created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(), updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() ); ALTER TABLE public.profiles ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users can read own profile" ON public.profiles FOR SELECT TO authenticated USING (id = auth.uid()); CREATE POLICY "Users can update own profile" ON public.profiles FOR UPDATE TO authenticated USING (id = auth.uid()); -- 自动创建用户资料 CREATE OR REPLACE FUNCTION public.handle_new_user() RETURNS TRIGGER LANGUAGE plpgsql SECURITY DEFINER AS $$ BEGIN INSERT INTO public.profiles (id, email, credits) VALUES (NEW.id, NEW.email, 0); RETURN NEW; END; $$; DROP TRIGGER IF EXISTS on_auth_user_created ON auth.users; CREATE TRIGGER on_auth_user_created AFTER INSERT ON auth.users FOR EACH ROW EXECUTE FUNCTION public.handle_new_user(); -- 2. Credit Transactions (积分交易记录) CREATE TABLE IF NOT EXISTS public.credit_transactions ( id UUID PRIMARY KEY DEFAULT gen_random_uuid(), user_id UUID NOT NULL REFERENCES auth.users(id), amount DECIMAL NOT NULL, type TEXT NOT NULL CHECK (type IN ('admin_recharge', 'purchase', 'consumption', 'refund')), description TEXT, created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() ); ALTER TABLE public.credit_transactions ENABLE ROW LEVEL SECURITY; CREATE POLICY "Users read own transactions" ON public.credit_transactions FOR SELECT TO authenticated USING (user_id = auth.uid()); -- 3. Admin Providers (管理员配置的服务商) CREATE TABLE IF NOT EXISTS public.admin_providers ( id TEXT PRIMARY KEY, provider_id TEXT NOT NULL, base_url TEXT NOT NULL, api_keys JSONB DEFAULT '[]'::JSONB, models JSONB DEFAULT '[]'::JSONB, created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(), updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() ); ALTER TABLE public.admin_providers ENABLE ROW LEVEL SECURITY; -- 4. Pricing Cache (价格缓存) CREATE TABLE IF NOT EXISTS public.provider_pricing_cache ( provider_id TEXT PRIMARY KEY, pricing JSONB NOT NULL DEFAULT '[]'::JSONB, cached_at TIMESTAMP WITH TIME ZONE DEFAULT NOW() ); ALTER TABLE public.provider_pricing_cache ENABLE ROW LEVEL SECURITY; CREATE POLICY "Allow read pricing cache" ON public.provider_pricing_cache FOR SELECT TO authenticated USING (true); -- 5. 管理员充值函数 CREATE OR REPLACE FUNCTION public.admin_recharge_credits(target_user_id UUID, amount DECIMAL, description TEXT DEFAULT '管理员充值') RETURNS VOID LANGUAGE plpgsql SECURITY DEFINER AS $$ BEGIN UPDATE public.profiles SET credits = COALESCE(credits, 0) + amount WHERE id = target_user_id; INSERT INTO public.credit_transactions (user_id, amount, type, description) VALUES (target_user_id, amount, 'admin_recharge', description); END; $$; -- 6. 消费积分函数 CREATE OR REPLACE FUNCTION public.consume_credits(amount DECIMAL, description TEXT DEFAULT '消费') RETURNS BOOLEAN LANGUAGE plpgsql SECURITY DEFINER AS $$ DECLARE current_credits DECIMAL; BEGIN SELECT credits INTO current_credits FROM public.profiles WHERE id = auth.uid(); IF current_credits < amount THEN RETURN FALSE; END IF; UPDATE public.profiles SET credits = credits - amount WHERE id = auth.uid(); INSERT INTO public.credit_transactions (user_id, amount, type, description) VALUES (auth.uid(), -amount, 'consumption', description); RETURN TRUE; END; $$;

4 验证配置

执行成功后，检查以下内容：

  • 点击左侧 Table Editor
  • 确认看到 profiles 表
  • 确认看到 credit_transactions 表
  • 确认看到 admin_providers 表
  • 确认看到 provider_pricing_cache 表
如果表不存在，请刷新页面重试

5 设置管理员账户

有两种方式设置管理员：

方式 A：通过邮箱标识（简单）

  1. 在 KK Studio 中注册一个用户
  2. 回到 Supabase Dashboard
  3. 点击 Table Editor → profiles
  4. 找到你的用户记录
  5. 将 email 修改为包含 @admin 的格式
    例如：yourname@admin.com

方式 B：指定具体邮箱（精确）

在 SQL Editor 中执行：

-- 修改为你自己的邮箱 UPDATE public.profiles SET email = 'your-email@example.com' WHERE id = 'your-user-id';

✓ 完成！
